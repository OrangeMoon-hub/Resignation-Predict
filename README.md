## 📊 模型性能

| 指标 | 数值 | 说明 |
|------|------|------|
| **召回率 (Recall)** | **0.74** | 74% 的真实离职者被提前预警 |
| **精确率 (Precision)** | 0.41 | 预警人群中 41% 确实会离职 |
| **F1-Score** | 0.51 | 召回与精确的调和平均 |

> ⚠️ 物流蓝领离职率天然偏低（正负样本极度不均衡），阈值策略主动偏向召回——**宁可多预警，不漏掉高风险员工。**

## 🧠 技术架构

```
┌──────────────────────────────────────────────────────────────────────┐
│                          数据层 (Data Layer)                          │
│  MySQL 多表联查 → 人效明细 ⋈ 薪资明细 ⋈ 仓库指标 ⋈ 离职记录            │
│  特征维度: 人事 / 经济 / 强度 / 稳定 四板块（含上下文特征）              │
└────────────────────────────────┬─────────────────────────────────────┘
                                 │
┌────────────────────────────────▼─────────────────────────────────────┐
│                       预处理 (Preprocessing)                           │
│  • Expanding Window 标准化/鲁棒标准化（逐期滚动，防数据偷窥）            │
│  • 样本重抽样：每期每人最多 3 条，防止时间维度过拟合                     │
│  • 缺值填补：岗位同期中位数填充综合评分缺失值                            │
└────────────────────────────────┬─────────────────────────────────────┘
                                 │
┌────────────────────────────────▼─────────────────────────────────────┐
│                      特征工程 (Feature Engineering)                    │
│  人事板块: 工龄 • 入职月份 • 岗位类型 • 仓库类型 • 潜力新员工             │
│  经济板块: 时薪(税后) • 收入排名 • 固浮比 • 基本工资拿满率 • 满勤率      │
│  强度板块: 考勤工时 • 标准工时 • 效率评分 • 协作工时占比                  │
│  稳定板块: 绩效增长率 • 收入增长率 • 过去6个月指标持续下降标记             │
│  筛选策略: 相关性 > 0.85 去冗余 + 随机森林累积重要性 Top 99%             │
└────────────────────────────────┬─────────────────────────────────────┘
                                 │
┌────────────────────────────────▼─────────────────────────────────────┐
│                      建模与评估 (Modeling)                             │
│  Balanced Random Forest (n=500)                                       │
│  + SMOTEENN 混合采样（过采样 + 欠采样）                                 │
│  + 样本权重 class_weight = {0:1, 1:4}                                 │
```

## 🔬 关键技术要点

### 1. 防数据偷窥 (Data Leakage Prevention)

| 手段 | 实现 |
|------|------|
| **Expanding Window 标准化** | 每期仅使用该期及之前的数据计算均值和标准差，模拟真实预测场景 |
| **重抽样策略** | 每期每人最多保留 3 条记录，防止同一员工的连续月份充当独立样本 |
| **滞后特征** | 过去 12 个月中位数、过去 6 个月下降标记等特征使用 `T-n` 窗口，不包含当前期 |

### 2. 样本不均衡处理

```
正负比 ≈ 1:12 → SMOTEENN + class_weight + 降低分类阈值
```

- **SMOTEENN**：先 SMOTE 过采样少数类，再 ENN 清洗边界噪声
- **Balanced Random Forest**：每棵树对多数类下采样，天然适应不均衡
- **阈值策略**：默认 0.5 → 调至 0.6 验证，PR 曲线辅助选最优平衡点

### 3. 特征降维

```
全量特征 (~50 维)
  → 相关性矩阵 (drop r > 0.85 的冗余特征对)
  → 随机森林特征重要性排序
  → 累积重要性 Top 99% 保留
  → 最终特征集 (~25 维)
```

### 4. 模型解释 (SHAP)

- **全局解释**：群体离职/留任 Top 5 驱动特征
- **局部解释**：逐样本 Top 7 影响因子 + 特征原值 + SHAP 归因值
- 结果写入 MySQL 结果表 + 影响因子表，供业务方查询
